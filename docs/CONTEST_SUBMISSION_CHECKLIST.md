## 3) 제출 전 품질 체크 (권장)

- **데이터 상태 점검**:

```bash
python src/data_audit.py
```

- **API 키 안전**:
  - `.env`는 제출물에 포함하지 않기
  - OpenAI API 호출 실패 시에도 리포트가 나오도록(현재 fallback 있음) 유지

- **PDF 20MB 제한**:
  - 스크린샷/이미지 수 최소화
  - 이미지 해상도/압축 조절(권장: 150~200 DPI)
  - 불필요한 중복 페이지/도표 제거

## 4) 권장 최종 PDF 구성(예시)

1. Executive Summary(요약)
2. 문제 정의(왜 필요한가)
3. 제안 서비스(누가/언제/어떻게 쓰는가, 사용자 플로우)
4. 추천 알고리즘(수치 기반 근거, 지표 정의, 계산식, 예시)
5. 데이터(제공/외부 구분, 전처리, 한계)
6. 데모(스크린샷 1~2장 + 실행 방법/영상 링크)
7. 기대 효과(스카우팅 비용 절감, 실패 영입 리스크 감소 등)
8. 한계 및 개선 로드맵(사업화 단계 포함)
9. 참고문헌/출처/라이선스(필수)
